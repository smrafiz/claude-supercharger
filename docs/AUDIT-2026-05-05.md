## Performance Issues

### 1. No hook-level caching (biggest win available)

[Benchmark source](https://github.com/disler/claude-code-hooks-multi-agent-observability):
- **50–100ms per full transcript parse** — applies to the observability project's complete history reconstruction
- **Supercharger context:** hooks read trimmed 20–50 line files → real improvement is **~5–10x**, not 50–100x
- **With 60s TTL cache:** ~1ms read, ~99% hit rate → confirmed for small-file reads

**Concurrency risk (MUST address):** Multiple Claude sessions share `$HOME/.claude/supercharger/scope/`. Cache keys must be session-scoped (e.g., `.{session_id}.cache`) to prevent cross-session data leakage. `tool-history-tracker` currently writes a flat `.tool-history` with no session namespace — concurrent sessions overwrite each other.

**Hooks that should cache (with session-scoped keys):**

| Hook | Cache key | TTL | Note |
|---|---|---|---|
| `tool-history-tracker` read | `{session_id}.tool-history` | 60s | Avoid re-parsing history. **Session-scoped required.** |
| `detect-stack` | `{cwd}.stack` | Session | Don't re-run git/package detection. |
| `repetition-detector` | `{session_id}.loop-history` | 60s | Already short, but session-scoped helps. |
| `session-memory-inject` | Already SessionStart-only | — | No TTL needed. |
| `agent-router` | Hash of prompt | 30s | Already has dedup — add TTL expiry only. |
| `config-scan` | `{project_dir}.config-scan` | Session | CLAUDE.md scan result stable. |

### 2. Process fork overhead (revised)

Async hooks with `async` flag do **not** block Claude — they run in the background after the tool completes.

Real sync PostToolUse count: ~4–5 (scope-guard check, repetition-detector, quality-gate, typecheck, confidence-gate). The "14 hooks" figure counted async hooks that don't block.

**Fix for process fork overhead:** Not a dispatcher that evaluates all concerns on every call (monolithic bottleneck risk). Instead: shared state file with incremental writes. Each hook reads only what it needs from a session-scoped state file, writes incrementally, exits fast.

### 3. No timeouts on hooks

[sdk best practice](https://github.com/gowaylee/cchooks): default 30s timeout, <100ms for fast safety checks.

`safety.sh` has a fast-path — Python fork only triggers on keyword-matched commands (~10% of calls). Add explicit timeout for the Python fork path.

### 4. UserPromptSubmit hook count (corrected)

**11 hooks**: 8 sync + 3 async. Not 13.

Sync (block prompt): `lesson-recall`, `agent-router`, `economy-reinforce`, `scope-guard contract`, `prompt-validator`, `reentry-detector`, `adaptive-economy`, `thinking-budget`

Async (don't block): `learn-from-prompts`, `context-advisor`, `rate-limit-advisor`

Many could merge into `prompt-preprocessor.sh` with fast-path early exits.

## Security Gap Analysis

### Current coverage vs alternatives

| Guardrail | Supercharger | claude-guard | agent-guardrails-template |
|---|---|---|---|
| Destructive Bash | Yes | Yes | Yes |
| Credential files (.env) via Read | Yes (`env-file-guard.sh`) | Partial | Yes |
| Force push to main | Yes | No | No |
| Prompt injection (MCP) | Yes | No | No |
| Shell wrapper hiding | Yes | Yes | No |
| Credential scanning output | Yes | No | No |
| Package manager enforcement | Yes | No | No |
| Git history rewriting | Partial | No | No |
| GitHub Actions injection | Yes (`code-security-scanner.sh:126-130`) | No | No |
| **Path traversal** | **No** | No | No |
| **Symlink attacks** | **No** | No | No |
| **Git hooks modification** | **No** | No | No |
| **Build artifact injection** | **No** | No | No |

### Missing security features

1. **Path traversal in Write/Edit paths**
   - Patterns: `../../../etc/passwd`, `%2e%2e%2f` (URL), `%252e%252e%252f` (double encode), null byte `%00` (path truncation on some filesystems), `foo/./bar/../baz` (normalized traversal), trailing `\0`
   - Check: normalize path (`realpath` or Python `os.path.normpath`), then verify resolved path under project root
   - Reference: [claude-code-hooks-mastery](https://github.com/disler/claude-code-hooks-mastery) docs

2. **Symlink attack detection**
   - **NOT** `[ -L "$FILE" ]` — only catches pre-existing symlinks
   - **Correct:** resolve full target path (`readlink -f` or Python `Path.resolve()`), verify under project root. Also covers symlinks created during the operation.

3. **Git hooks modification detection**
   - Block edits under: `.git/hooks/`, `.githooks/`, `$(git rev-parse --git-common-dir)/hooks/`, `$HOME/.claude/hooks/` (supercharger hooks — compromise here = disable all security checks)
   - Pattern: pre-commit, pre-push, post-commit hooks

4. **Build artifact injection**
   - Pattern: modifying `node_modules/.bin/`, `__pycache__/`, `.next/`, `.venv/`, `vendor/` — injects trojanized binaries that execute in build scripts
   - Mechanism: **dependency trojaning**, not path traversal

5. **Absolute-path writes outside project root** *(missing from original audit)*
   - Write/Edit to `~/.ssh/`, `~/.aws/`, `~/.config/`, `~/.npmrc`, `~/.gitconfig` — credential persistence without traversal
   - Pattern: absolute paths that resolve outside project root

6. **Named pipe / device file writes** *(missing from original audit)*
   - Pattern: write to `/dev/stdin`, `/dev/stdout`, `/proc/self/fd/*`, `/tmp` symlink races
   - Check: reject writes to known device paths

7. **Git reflog / object manipulation** *(missing from original audit)*
   - Pattern: `.git/refs/`, `.git/objects/` manipulation, reflog rewriting
   - Already covers force-push; extend to ref-level control

## Token Economy Analysis

### What's good

- Stack standards injection at session start (tier-scaled)
- Economy tier enforcement with `economy-reinforce.sh`
- MCP profile context management
- Session cost tracking + budget caps

### What's missing

1. **Session context injection** — [claude-hooks-sdk pattern](https://github.com/hgeldenhuys/claude-hooks-sdk): inject session ID + git branch + time on every UserPromptSubmit. Saves 50–100 tokens per request by avoiding tool calls.

2. **Hook output suppression** — `suppressOutput: true` in hook JSON prevents noise from entering context. Currently only `confidence-gate.sh` uses it.

3. **Prompt cache awareness** — stack standards reinject on every `SessionStart` regardless of cache hit. Could gate on cache hit rate.

4. **Adaptive economy thresholds** — `adaptive-economy.sh` exists but could be more aggressive.

- **Confidence gate** — only project with observable-signal-based scoring

- **Hook-level budget caps** — PreToolUse budget enforcement

- **Override-feedback loop** — counts repeated disables, suggests `.supercharger.json` exceptions

### Areas where competitors are better

- TypeScript SDK → better testing, composability, type safety (all Supercharger hooks are raw bash)
- LLM-based prompts → `cchooks` uses LLM evaluation for complex security decisions where regex fails
- Session context injection → 50–100 tokens saved per prompt
- Per-tool rejection chains → suggest alternatives rather than blanket deny
